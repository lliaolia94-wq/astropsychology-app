# Временная заглушка для пропуска проверки SMS

## Проблема

Для тестирования регистрации пользователей требуется подтверждение SMS-кода, но в тестовом окружении это не всегда удобно.

## Решение

Добавлена временная заглушка, которая позволяет пропустить проверку SMS-кода через переменную окружения.

## Как включить

### Вариант 1: Через .env файл (рекомендуется)

Создайте или откройте файл `.env` в корне проекта и добавьте:

```env
SKIP_SMS_VERIFICATION=true
```

### Вариант 2: Через переменную окружения системы

**Windows (PowerShell):**
```powershell
$env:SKIP_SMS_VERIFICATION="true"
```

**Windows (CMD):**
```cmd
set SKIP_SMS_VERIFICATION=true
```

**Linux/Mac:**
```bash
export SKIP_SMS_VERIFICATION=true
```

### Вариант 3: При запуске сервера

```bash
SKIP_SMS_VERIFICATION=true python main.py
```

## Как отключить

Установите значение в `false` или удалите переменную окружения:

```env
SKIP_SMS_VERIFICATION=false
```

Или просто удалите строку из `.env` файла.

## Важно

⚠️ **Эта заглушка предназначена ТОЛЬКО для тестирования!**

- Не используйте в production
- Не коммитьте `.env` с `SKIP_SMS_VERIFICATION=true` в репозиторий
- Добавьте `.env` в `.gitignore`, если его там нет

## Где работает

Заглушка работает для следующих эндпоинтов:

1. `/auth/register` - регистрация пользователя
2. `/auth/reset-password-confirm` - подтверждение сброса пароля

## Проверка

После включения заглушки, при регистрации в логах сервера вы увидите:

```
⚠️ ПРОВЕРКА SMS ОТКЛЮЧЕНА (тестовый режим). Регистрация пользователя +79161234567 без проверки SMS-кода
```

Теперь можно регистрировать пользователей без подтверждения SMS-кода.
